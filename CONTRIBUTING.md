# Contributing to cardano-mcp

Thanks for considering a contribution. This doc covers the moving parts you'll touch.

## Setup

```sh
pnpm install
pnpm build
pnpm test
```

Tests are pure unit tests — no provider credentials needed.

## CI checks

Every PR must pass all five locally before push:

```sh
pnpm lint && pnpm format:check && pnpm typecheck && pnpm build && pnpm test
```

If formatting fails: `pnpm format` then commit the result.

## Adding a new tool — four touch points

Every new tool must be reflected in **all** of these:

1. **Tool definition** — JSON schema in `src/tool-definitions.ts`. This is the only doc the AI reads, so spend time on it. Use enums for finite choices, mark required parameters, document defaults inline.
2. **Handler** — function in `src/handlers/<domain>-handlers.ts`. Pattern: `normalizeParameters` → validate → call provider → wrap response.
3. **Router** — entry in `HANDLER_MAP` in `src/tool-router.ts`.
4. **Tests** — validation-path tests in `test/unit/handlers.test.ts`.

Also update the tool table in `README.md` and tick off the corresponding item in `TODO.md`.

### Handler skeleton

```ts
import type { ServerContext } from "../context.js";
import type { ToolResponse } from "../types.js";
import {
  createErrorResponse,
  jsonResponse,
  normalizeParameters,
} from "../utils.js";
import { ProviderError, getProvider } from "../providers/index.js";

export async function handleYourTool(
  ctx: ServerContext,
  rawArgs: any,
): Promise<ToolResponse> {
  const args = normalizeParameters(rawArgs);

  if (!args.requiredThing) {
    return createErrorResponse("requiredThing is missing", [
      "Provide a non-empty value",
    ]);
  }
  // ... other validators

  try {
    const provider = getProvider(ctx, {
      provider: args.provider,
      network: args.network,
    });
    const result = await provider.yourMethod(args.requiredThing);
    return jsonResponse(result);
  } catch (err) {
    if (err instanceof ProviderError) {
      return createErrorResponse(err.message);
    }
    return createErrorResponse(
      `Unexpected error: ${err instanceof Error ? err.message : String(err)}`,
    );
  }
}
```

## Adding a new provider backend

1. Implement `CardanoProvider` from `src/providers/base.ts`. Methods you can't support must `throw new ProviderError("not_supported", "...")`.
2. Register the backend in `src/providers/index.ts` — add to the `getProvider` switch and the `ProviderName` union in `src/types.ts`.
3. Add the new option to the `provider` enum in `tool-definitions.ts`.
4. Update the provider-coverage matrix in `README.md` and `CLAUDE.md`.

## Style

- TypeScript strict mode + `noUncheckedIndexedAccess`.
- Errors as values, not exceptions, at the handler boundary — that's the whole point of `ToolResponse.isError`.
- Don't add abstractions that aren't earned. Three similar handlers is fine; the moment they get genuinely duplicative, factor them out.
- Comments answer **why**, not what. Skip them when the code is self-explanatory.

Open an issue on GitHub — happy to discuss design before you start coding anything substantial.
